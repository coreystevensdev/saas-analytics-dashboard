## What Went Well

### Architecture Patterns Held
- SWR fallbackData pattern (Story 2.6) — Server Component fetches initial data, passes to Client as SWR fallback. Zero loading state on first render. This becomes the standard for the rest of the project.
- HMAC preview token (Story 2.3) — Stateless TOCTOU protection without Redis. SHA-256 file hash + org ID + timestamp with timing-safe comparison.
- Product-blame error messages (Story 2.2) — "We couldn't read..." never "You uploaded a bad file." NFR21 done right.
- Seed data hybrid global + transparent fallback (Story 2.1) — One dedicated `seed-demo` org, no per-org copies.

## What Didn't Work

### jsdom 28 + user-event Incompatibility
`FileList.item()` removed in jsdom 28 broke `@testing-library/user-event` file upload tests. Required switching all `userEvent.upload()` calls to `fireEvent.change` with manual `files` property assignment. Hit in Stories 2.2 and 2.3.

### React 19 Lint Rule Surprises
`react-hooks/set-state-in-effect` and `react-hooks/refs` rules forced refactors in Stories 2.6 and 2.8. The fix pattern (`useSyncExternalStore` or `useState`-based render-time adjustment) is now well understood but cost time discovering.
