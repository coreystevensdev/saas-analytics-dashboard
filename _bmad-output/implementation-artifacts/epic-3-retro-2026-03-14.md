## Successes

1. **Layered pipeline architecture** — Each story built cleanly on the previous. The computation → scoring → assembly → streaming → subscription → transparency layering worked exactly as designed. No story required rework of previous stories' code.

2. **Pure-function design** — `computeStats()`, `scoreInsights()`, `assemblePrompt()`, `streamReducer()` — all testable without mocking databases or network calls. This decision in Story 3.1 paid dividends through Story 3.6. Exported pure functions became the team's preferred testing pattern.

3. **Privacy-by-architecture enforcement** — TypeScript types prevented `DataRow[]` from reaching the LLM. This wasn't convention — the compiler enforced it. `assembly.ts` accepts `ScoredInsight[]`, period. Held firm across all 6 stories.

4. **Adversarial code review caught a critical bug** — Story 3.6's review discovered that `validatedMetadata` was never included in the `partial` SSE event. The reducer no-ops `DONE` after `PARTIAL`, so even the trailing done event couldn't deliver metadata. Users in timeout state would have seen an empty transparency panel. Three-layer fix: shared type → backend event → frontend reducer.

5. **a11y-first development** — Consistently strong across all stories. Story 3.6 had `aria-live`, `aria-expanded`, keyboard Escape handling, focus management, 44x44px touch targets, `motion-reduce:duration-0`. This was an Epic 2 retro action item that the team successfully adopted.

6. **Metadata convergence pattern** — Two completely different data paths (SSE stream for authenticated users, RSC cache for anonymous) converging at `DashboardShell` via nullish coalescing (`streamMetadata ?? cachedMetadata ?? null`). Clean architecture for a hard problem.

7. **`useSyncExternalStore` pattern** — Applied correctly in `useIsMobile` and `useReducedMotion`. No hydration mismatches. The Epic 2 retro's documentation of React 19 patterns paid off here.

## Challenges

1. **Test boilerplate** — Every integration test needed `vi.mock('../../config.js')` + `vi.mock('../../lib/logger.js')` because logger imports config which validates env vars at import time. ~5 lines of copy-paste per test file across 10+ files.

2. **shadcn not configured** — Story 3.6 discovered the project lacks `components.json`. `npx shadcn@latest add sheet` failed. Worked around with native `<dialog>` BottomSheet. This created a split component library — part shadcn primitives, part hand-rolled.

3. **Pre-existing broken checks** — `shared` package vitest and `web:lint` have been broken since before Epic 3. Not showstoppers, but they erode confidence in the test suite and prevent verification of shared package changes.

4. **SSE event path coverage gap** — Tests covered `DONE` metadata and `CACHE_HIT` metadata but not `PARTIAL` metadata. The adversarial review caught the bug, not the test suite. Revealed that terminal-state testing needs to be systematic.

5. **Mock hoisting issues** — `vi.mock` factory must define all constants inline because Vitest hoists above variable declarations. Hit this in Stories 3.1, 3.2, 3.3 before the pattern became established.

6. **Story 3.2 file status** — Still shows "review" instead of "done". Minor housekeeping gap.

## Key Insights

1. **Cross-layer bugs need adversarial review** — Unit tests catch single-layer bugs. The timeout metadata bug spanned shared types, backend events, and frontend reducer. Only a human reviewer asking "what happens in the timeout path?" caught it.

2. **Pure-function exports are the team's best testing pattern** — `streamReducer`, `computeStats`, `scoreInsights`, `assemblePrompt` — all exported, all tested in isolation. Repeat this pattern.

3. **Every terminal state needs dedicated tests** — Happy path testing isn't enough. Each SSE event type (`DONE`, `CACHE_HIT`, `PARTIAL`, `ERROR`, `UPGRADE_REQUIRED`) should have tests for every field it carries.

4. **Tech debt carried across epics compounds** — The unbounded charts query from Epic 2 now affects Epic 4's PNG rendering design. The broken shared tests prevent verification of shared package changes. Debt doesn't stay static — it grows.
